## Goal
Implement the transaction model types: Item, Post, Xact (Transaction), Journal, Balance, and Value.

## Acceptance Criteria
- [x] Item base type with date, state (cleared/pending/uncleared), note, tags
- [x] Post type with account, amount, cost, flags (virtual, balance-virtual, generated)
- [x] Xact (Transaction) type with date, payee, posts, finalize() for balance checking
- [x] Journal type that holds list of transactions and account tree
- [x] Balance type: multi-commodity balance (dictionary of commodity → amount)
- [x] Value type: polymorphic wrapper (empty, Amount, or Balance) with arithmetic
- [x] Transaction finalize() validates postings balance (single commodity sums to zero)
- [x] Null amount inference: if one posting has no amount, infer from remainder
- [x] Multi-commodity transactions: when all postings have explicit amounts in different commodities, treat as implicit exchange (skip zero-sum check)
- [x] Unit tests for Balance arithmetic and transaction finalization

### Key finalize() logic (from Python xact.py):
1. Separate postings into groups: real, balanced-virtual `[Account]`, virtual `(Account)`
2. For each group, sum amounts
3. If exactly one posting has null amount → infer it from remainder
4. If balance is multi-commodity (Value::Balance) and no null posting → treat as implicit exchange (skip check)
5. If single commodity and sum ≠ 0 → error "Transaction does not balance"

### Post flags needed for parity:
- `is_virtual` — `(Account)` parentheses
- `is_balance_virtual` — `[Account]` brackets
- `has_cost` — `@ price` or `@@ total`
- `cost` — the cost Amount
- `assigned_amount` — for balance assertions `= $X`
- `lot_annotation` — `{$price}` lot annotation

### State enum:
- Uncleared (default)
- Cleared (`*`)
- Pending (`!`)
