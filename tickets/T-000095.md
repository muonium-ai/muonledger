## Goal
Implement the transaction model types: Item, Post, Xact (Transaction), Journal, Balance, and Value.

## Acceptance Criteria
- [ ] Item base class with date, state (cleared/pending/uncleared), note, tags
- [ ] Post class with account, amount, cost, flags (virtual, balanceVirtual, generated)
- [ ] Xact (Transaction) class with date, payee, posts, finalize() for balance checking
- [ ] Journal class that holds list of transactions and account tree
- [ ] Balance class: multi-commodity balance (Map<String, Amount>)
- [ ] Value sealed class: polymorphic wrapper (Empty, AmountValue, BalanceValue) with arithmetic
- [ ] Transaction finalize() validates postings balance
- [ ] Null amount inference: if one posting has no amount, infer from remainder
- [ ] Multi-commodity: implicit exchange when all postings have explicit amounts in different commodities
- [ ] Unit tests for Balance arithmetic and transaction finalization

### Kotlin-specific notes:
- Use `sealed class Value` with `data class AmountValue(val amount: Amount)`, `data class BalanceValue(val balance: Balance)`, `data object Empty`
- Use `enum class State { UNCLEARED, CLEARED, PENDING }`
- Post flags can use boolean properties or an EnumSet

### Key finalize() logic:
1. Separate postings into groups: real, balanced-virtual `[Account]`, virtual `(Account)`
2. For each group, sum amounts
3. If exactly one posting has null amount → infer it from remainder
4. If balance is multi-commodity and no null posting → implicit exchange (skip check)
5. If single commodity and sum ≠ 0 → error
