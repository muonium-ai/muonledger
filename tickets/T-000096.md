## Goal
Implement the textual journal parser that reads ledger-format files and produces a Journal with transactions.

## Acceptance Criteria
- [ ] Parse transaction headers: `YYYY-MM-DD [*|!] Payee [| Note]`
- [ ] Parse postings: `    Account  [$]Amount[.cents]`
- [ ] Parse null amounts (inferred posting)
- [ ] Parse negative amounts: `$-15.00`
- [ ] Parse date with both `-` and `/` separators
- [ ] Parse commodity amounts: `100 EUR`, `$50.00`, `10 AAPL`
- [ ] Parse large amounts with comma separator: `$12,500.00`
- [ ] Parse inline posting comments: `$25.00  ; lunch`
- [ ] Parse transaction-level comments/metadata: `; Payee: Staples`
- [ ] Parse virtual postings: `(Budget:Food)` and `[Budget:Emergency]`
- [ ] Parse cost annotations: `@ $1.10` and `@@ $110.00`
- [ ] Parse lot annotations: `{$150.00}` → derive cost = qty * price
- [ ] Parse balance assertions: `= $950.00`
- [ ] Parse directives: `alias`, `apply account`/`end apply account`, `comment`/`end comment`, `include`, `P` price
- [ ] Parse automated transactions: `= Predicate`
- [ ] Parse periodic transactions: `~ Period`
- [ ] Handle file inclusion: `include other.dat`
- [ ] Parser produces valid Journal with finalized transactions
- [ ] Unit tests for parsing various journal formats

### Parser structure (line-by-line):
The parser reads the file line by line:

1. **Blank lines** → end current transaction context
2. **Lines starting with digit** → transaction header: `YYYY-MM-DD [*|!] Payee`
3. **Lines starting with whitespace** → posting within current transaction
4. **`alias X=Y`** → account alias mapping
5. **`apply account X`** → push prefix onto account stack
6. **`end apply account`** → pop prefix from account stack
7. **`comment`** → skip until `end comment`
8. **`include FILE`** → recursively parse another file
9. **`P DATE COMMODITY PRICE`** → price directive (store in journal)
10. **`= PREDICATE`** → automated transaction
11. **`~ PERIOD`** → periodic transaction
12. **Lines starting with `;` or `#`** → comment, skip

### Posting parsing:
```
    Account  Amount [@ CostPerUnit | @@ TotalCost] [{LotPrice}] [= AssertAmount] [; Comment]
```

When parsing amount:
- Commodity symbol can be prefix (`$50`) or suffix (`100 EUR`)
- Detect commodity style from first occurrence
- Strip comma thousands separators before parsing number

### Lot annotation cost derivation:
When `{$150.00}` is parsed and no explicit `@ cost`:
```
cost = |quantity| * lot_price
```

### apply account:
Maintain a stack of prefixes. When `apply account Assets` is active, account `Checking` becomes `Assets:Checking`.
