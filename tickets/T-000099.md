## Notes

### CRITICAL: Fix parity test Kotlin invocation
The current test_parity.py builds commands as:
```python
cmd = list(impl.cmd) + ["-f", journal_path, test.command] + test.args
```
For Kotlin this produces: `./gradlew run --args -f test.dat balance`
But Gradle's `--args` requires a SINGLE quoted string: `./gradlew run --args="-f test.dat balance"`

**You MUST update `port/scripts/test_parity.py`** to handle Kotlin specially. In the `run_single_test()` function, when the implementation name is "kotlin", join the arguments into a single string for `--args=`.

Also update the kotlin Implementation definition to:
```python
"kotlin": Implementation(
    name="kotlin",
    cmd=["./gradlew", "run", "--args"],
    cwd=str(PORT_DIR / "kotlin"),
    check_cmd=["./gradlew", "--version"],
),
```

And in `run_single_test()`, add logic like:
```python
if impl.name == "kotlin":
    args_str = " ".join(["-f", journal_path, test.command] + test.args)
    cmd = list(impl.cmd)[:-1] + [f"--args={args_str}"]
else:
    cmd = list(impl.cmd) + ["-f", journal_path, test.command] + test.args
```
