## Goal
Set up the Swift package and implement core data types: Amount, Commodity/CommodityPool, and Account tree.

## Acceptance Criteria
- [ ] `port/swift/Package.swift` exists with executable target `muonledger`
- [ ] `swift build` succeeds in `port/swift/`
- [ ] Amount type with exact decimal arithmetic (use Foundation's Decimal or a Fraction type)
- [ ] Commodity type with symbol, precision, and formatting styles
- [ ] CommodityPool for commodity registry/interning
- [ ] Account type with hierarchical parent/children tree and fullname
- [ ] Unit tests for Amount arithmetic (+, -, *, negate, comparison)

## Notes
Reference implementations:
- `port/python/src/muonledger/amount.py` — Amount with fractions.Fraction for exact math
- `port/python/src/muonledger/commodity.py` — Commodity with styles (prefix/suffix symbol, precision)

### Amount requirements
- Exact decimal arithmetic (no floating point)
- Commodity association (amount has a commodity)
- Formatting: respects commodity style (prefix `$100.00` vs suffix `100 EUR`)
- Parsing: `$50.00`, `100 EUR`, `$-15.00`, `$12,500.00` (thousands separator)
- Operations: add, subtract, multiply by scalar, negate, abs, is_zero, comparison

### Commodity requirements
- Symbol (e.g., `$`, `EUR`, `AAPL`)
- Precision (decimal places)
- Style flags: prefix vs suffix, thousands separator, decimal comma
- CommodityPool: find_or_create(symbol) → returns same instance

### Account requirements
- Name and fullname (e.g., `Checking` with fullname `Assets:Checking`)
- Parent/children tree
- find_or_create_account("Assets:Checking:Savings") creates hierarchy
- depth() method
