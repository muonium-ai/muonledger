## Goal
Allow multi-commodity transactions to balance when different commodities are present without explicit `@` cost annotations, matching C++ ledger behavior.

## Acceptance Criteria
- [x] Python: `100 EUR` + `$-110.00` in same transaction balances (no error)
- [x] Rust: Same transaction balances
- [x] Python: `P` price directive journal parses without error
- [x] Rust: `P` price directive journal parses without error
- [x] Parity tests `multi_commodity` and `price_directive` pass for both

## Notes
C++ ledger allows multi-commodity transactions without explicit `@` cost — it just tracks each commodity separately. Both Python and Rust currently reject these as "Transaction does not balance" because they compare across commodities.

The fix: when checking balance, group amounts by commodity. If all commodities individually have at most one posting (or net to zero within their commodity), the transaction is considered balanced. Alternatively, when there are exactly 2 commodities, treat it as an implicit exchange.
