## Goal
Implement the textual journal parser that reads ledger-format files and produces a Journal with transactions.

## Acceptance Criteria
- [ ] Parse transaction headers: `YYYY-MM-DD [*|!] Payee [| Note]`
- [ ] Parse postings with amounts, null amounts, negative amounts
- [ ] Parse date with both `-` and `/` separators
- [ ] Parse commodity amounts: `100 EUR`, `$50.00`, `10 AAPL`, `$12,500.00`
- [ ] Parse inline posting comments and transaction metadata
- [ ] Parse virtual postings: `(Account)` and `[Account]`
- [ ] Parse cost: `@ $1.10`, `@@ $110.00`
- [ ] Parse lot annotations: `{$150.00}` → derive cost = qty * price
- [ ] Parse balance assertions: `= $950.00`
- [ ] Parse directives: `alias`, `apply account`/`end apply account`, `comment`/`end comment`, `include`, `P` price
- [ ] Parse automated transactions `=` and periodic transactions `~`
- [ ] Handle file inclusion: `include other.dat`
- [ ] Parser produces valid Journal with finalized transactions
- [ ] Unit tests

## Notes
Reference implementations:
- `port/python/src/muonledger/parser.py` — Full-featured textual parser (~1500 lines)
- `port/rust/src/parser.rs` — Rust parser (~1200 lines)

### Parser structure (line-by-line state machine):
1. **Blank lines** → end current transaction
2. **Digit-starting** → transaction header: `YYYY-MM-DD [*|!] Payee`
3. **Whitespace-starting** → posting in current transaction
4. **`alias X=Y`** → account alias
5. **`apply account X`** / **`end apply account`** → prefix stack
6. **`comment`** / **`end comment`** → block comment
7. **`include FILE`** → recursive parse
8. **`P DATE COMMODITY PRICE`** → price directive
9. **`= PREDICATE`** → automated transaction
10. **`~ PERIOD`** → periodic transaction
11. **`;` or `#`** → line comment

### Posting format:
```
    Account  Amount [@ Cost | @@ TotalCost] [{LotPrice}] [= AssertAmt] [; Comment]
```

### Lot annotation cost derivation:
`{$150.00}` with no explicit `@`: derive `cost = |quantity| * lot_price`

### apply account:
Stack-based prefix. `apply account Assets` → `Checking` becomes `Assets:Checking`.
