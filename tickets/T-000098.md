### CLI argument parsing:
Use Swift's `ArgumentParser` package (swift-argument-parser) or manual argv parsing.
```
muonledger -f <file> <command> [options] [patterns...]
muonledger --file <file> <command> [--begin DATE] [--end DATE] [patterns...]
muonledger --version
```

### Parity test invocation:
The parity test system calls: `swift run muonledger -f test.dat balance [args...]`

### Balance output format (must match C++ ledger):
```
             $200.00  Assets
             $200.00    Checking
            $-200.00    Savings
--------------------
                   0
```
Key formatting rules:
- Amount right-aligned in ~20 char column
- Account indented by 2 spaces per depth level from display root
- Accounts with 0 balance and single child are collapsed (e.g., `Assets:Checking` instead of `Assets\n  Checking`)
- When parent is collapsed, children show full path from collapsed parent
- Separator line of dashes
- Grand total at bottom

### Register output format:
```
24-Jan-15 Grocery Store          Expenses:Food                $50.00       $50.00
                                 Assets:Checking             $-50.00            0
```
Key formatting rules:
- Date in YY-Mon-DD format (e.g., `24-Jan-15`)
- Payee truncated/padded to ~20 chars
- Account padded to ~25 chars
- Amount right-aligned
- Running total right-aligned (cumulative sum of all postings so far)

### Print output format:
```
2024-01-15 Grocery Store
    Expenses:Food              $50.00
    Assets:Checking           $-50.00
```

### Date filtering:
- `--begin DATE`: include transactions where xact.date >= parsed_date
- `--end DATE`: include transactions where xact.date < parsed_date
- Date format: YYYY-MM-DD

### Account filter:
- Positional args after command are account filter patterns
- Simple substring match on account fullname is sufficient for parity
