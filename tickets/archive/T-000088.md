## Goal
Fix `apply account` directive to properly prefix account names in both Python and Rust.

## Acceptance Criteria
- [x] Python: `apply account Assets` causes `Checking` to become `Assets:Checking` in balance output
- [x] Rust: Same behavior
- [ ] Parity test `apply_account` passes for both

## Notes
Both Python and Rust parse `apply account` but don't prefix the accounts — output shows `Checking` instead of `Assets:Checking`. The account prefix should be prepended to all account names in transactions between `apply account X` and `end apply account`.
