### Important: CLI invocation pattern
The parity test system invokes Kotlin as: `./gradlew run --args "-f test.dat balance"`
This requires the `application` plugin with `mainClass` set. Use:
```kotlin
plugins {
    kotlin("jvm") version "2.0.0"
    application
}
application {
    mainClass.set("muonledger.MainKt")
}
```

BUT the parity test currently builds the command as:
```python
cmd = ["./gradlew", "run", "--args"] + ["-f", journal_path, test.command] + test.args
```
This won't work because `--args` needs a single quoted string. We need to ALSO update `port/scripts/test_parity.py` to fix the Kotlin invocation. Change it to build args as a single string:
```python
"kotlin": Implementation(
    name="kotlin",
    cmd=["./gradlew", "run", "--args"],
    ...
)
```
And in the test runner, join args for Gradle: if impl is kotlin, `cmd = ["./gradlew", "run", "--args", " ".join(["-f", journal_path, test.command] + test.args)]`

### Amount requirements
- Use `java.math.BigDecimal` for exact decimal arithmetic
- Commodity association
- Formatting: prefix `$100.00` vs suffix `100 EUR`
- Parsing: `$50.00`, `100 EUR`, `$-15.00`, `$12,500.00`
- Operations: add, subtract, multiply, negate, abs, isZero, compareTo

### Commodity requirements
- Symbol, precision, style flags (prefix/suffix, thousands separator)
- CommodityPool: findOrCreate(symbol)

### Account requirements
- Name, fullname (`Assets:Checking`)
- Parent/children tree
- findOrCreateAccount("Assets:Checking") creates hierarchy
