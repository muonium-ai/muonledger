## Goal
Implement the textual journal parser that reads ledger-format files and produces a Journal with transactions.

## Acceptance Criteria
- [x] Parse transaction headers: `YYYY-MM-DD [*|!] Payee [| Note]`
- [x] Parse postings with amounts, null amounts, negative amounts
- [x] Parse date with both `-` and `/` separators
- [x] Parse commodity amounts: `100 EUR`, `$50.00`, `10 AAPL`, `$12,500.00`
- [x] Parse inline posting comments and transaction metadata
- [x] Parse virtual postings: `(Account)` and `[Account]`
- [x] Parse cost: `@ $1.10`, `@@ $110.00`
- [x] Parse lot annotations: `{$150.00}` → derive cost = qty * price
- [x] Parse balance assertions: `= $950.00`
- [x] Parse directives: `alias`, `apply account`/`end apply account`, `comment`/`end comment`, `include`, `P` price
- [x] Parse automated transactions `=` and periodic transactions `~`
- [x] Handle file inclusion: `include other.dat`
- [x] Parser produces valid Journal with finalized transactions
- [x] Unit tests

### Parser structure (line-by-line state machine):
1. **Blank lines** → end current transaction
2. **Digit-starting** → transaction header: `YYYY-MM-DD [*|!] Payee`
3. **Whitespace-starting** → posting in current transaction
4. **`alias X=Y`** → account alias
5. **`apply account X`** / **`end apply account`** → prefix stack
6. **`comment`** / **`end comment`** → block comment
7. **`include FILE`** → recursive parse
8. **`P DATE COMMODITY PRICE`** → price directive
9. **`= PREDICATE`** → automated transaction
10. **`~ PERIOD`** → periodic transaction
11. **`;` or `#`** → line comment

### Posting format:
```
    Account  Amount [@ Cost | @@ TotalCost] [{LotPrice}] [= AssertAmt] [; Comment]
```

### Lot annotation cost derivation:
`{$150.00}` with no explicit `@`: derive `cost = |quantity| * lot_price`

### apply account:
Stack-based prefix. `apply account Assets` → `Checking` becomes `Assets:Checking`.
