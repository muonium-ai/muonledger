## Goal
Fix Rust lot annotation parsing so that `10 AAPL {$150.00}` derives an implicit cost of `$1500.00` for transaction balancing.

## Acceptance Criteria
- [ ] `10 AAPL {$150.00}` + `$-1500.00` balances without error
- [ ] Parity test `lot_price` passes for Rust

## Notes
Rust parses `{$150.00}` as lot annotation but doesn't derive `cost = quantity * lot_price` for balance checking. Same fix was applied to Python in T-000081. Error: "Transaction does not balance: remainder is $-1500.00 10 AAPL {$150.00}".
