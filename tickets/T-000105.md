## Goal
Make `check_and_commit()` return truthful commit status and avoid mutating `best.tsv` as a side effect when git commit fails.

## Acceptance Criteria
- [ ] `check_and_commit()` returns `False` when `commit_best()` fails or returns `None`
- [ ] Console output never claims "committed to git" unless the commit actually succeeded
- [ ] `best.tsv` is not written until the git commit is confirmed, or changes are rolled back on failure
- [ ] Consider making auto-commit opt-in rather than default
